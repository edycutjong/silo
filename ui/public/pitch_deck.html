Slide 02 // The Problem

Anonymity is Broken in Web2 Leaks

Identity Leaks

Every standard upload form logs IP addresses, browser fingerprints, and request payloads, leaving a clear path to the whistleblower.

OTP Trap

Verifying that a source controls a real, domain-specific contact details (Insider verification) traditionally logs their plaintext phone/email.

Slide 03 // The Solution

Hardware-Airgapped Credibility

Silo moves the entire dropping and verification logic into a **secure hardware enclave**. Plaintext PII is used inside the TEE to send OTP codes and verify integrity, but *never* hits databases, logs, or agent memory.

100% Enclave Verification Zero PII Logs
Slide 04 // Core Product Flow

How Silo Operates

01. DROP

Evidence PDF is uploaded and hashed directly inside the enclave.

02. OTP CHALLENGE

Whistleblower receives an SMS OTP code generated by T3.

03. MANIFEST SIGN

Enclave verifies code, signs report manifest VC, and clears OTP.

04. DISPATCH

Report dispatches to media outbox via blind relay placeholder.
